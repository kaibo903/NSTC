PRD：地主都市更新全案管理試算平台（簡單直覺版｜No DB｜MVP）
1. 產品目標與定位

本平台為「地主自建開發收益評估資訊系統」之 MVP，提供地主在都市更新初期，以最少輸入、最直覺的操作方式，完成五大構面試算並輸出評估報告。平台不採用資料庫、不提供帳號與歷史版本；使用者完成輸入後即可立即得到結果，成果保存以 PDF/Excel 匯出與 JSON 匯入/匯出為主。

核心目標

讓地主在 10–15 分鐘內完成一案的初步試算

每一步都能看到「下一步要做什麼」與「目前結果代表什麼」

每個核心指標可點選「查看依據」顯示定義/公式/來源（靜態引用庫）

2. 目標使用者與使用情境
2.1 Persona

地主一般使用者（主要）：不熟法規與工程估算，只想快速知道「這案子大概划不划算」「可以分回多少」「共同負擔高不高」。

2.2 使用情境（MVP）

單一案件、單一情境：輸入基地條件與假設參數 → 試算 → 下載報告 → 再調整幾個假設重新試算。

3. 產品原則（簡單直覺）

單一路徑：使用者永遠只要「從上到下填」即可完成。

少填少錯：能預設的都預設，必填欄位最少化，進階欄位收合。

即時回饋：一改數值，右側立即更新關鍵結果摘要（不需跳頁）。

錯誤提示要可操作：告訴使用者「哪個欄位」「為什麼」「怎麼改」。

結果先講白話：先顯示結論與重要數字，再展開明細。

不使用資料庫：不儲存使用者資料；用匯出檔案保留成果。

4. 操作流程（User Flow）

平台採單頁 Stepper（或上方分頁 Tabs），固定 5 步：

基本資料（輸入）

建築面積試算（自動、唯讀）

銷售面積試算（少量可調）

成本費用（少量可調）

評估報告（核心指標＋解釋/決策＋匯出）

每一步頁面底部固定三個操作按鈕（位置一致）：

上一頁（第一步不顯示）

下一步

重新計算（每一步都可用；按下即更新右側摘要）

頂部固定兩個按鈕：

匯出 PDF

匯出 Excel
（只有當必填欄位通過驗證後才啟用）

另提供：

匯出 JSON（輸入+輸出）

匯入 JSON（回填欄位）
作為無資料庫情境下的「保存/重做」方式。

6. 參數類型（必須出現在產品內）

平台需在「說明區」呈現參數類型定義（可放在第一步頁首）：

個案參數：依基地基本資料輸入（基地面積、建蔽率、容積率、更新前原室內坪數、地主人數等）

固定參數：依法規或經驗值之固定值（機電上限、梯廳上限、陽台上限、單層屋突上限、費率％）

變動參數：假設性較高之參數（預估公設比、車位單位面積、折減係數、銷售面積百分比）

7. 功能需求（五大構面）
7.1 Step1：基本資料（Input）

必填（MVP 最小集合）

基地面積（m²）

建蔽率（%）

容積率（%）

開挖率（%）

最寬道路寬度（m）

預估地上單層樓高（m，預設 3.2）

預估屋突層數（預設 2）

預估地下層數（預設 3）

更新前原室內坪數（坪）

地主人數（人）

營建成本（元/坪）

法定工程造價（元/m²）

選填（進階收合）

地號、地段

使用分區

最大路寬至建築物深度

固定參數（顯示但預設不可改；如要可改放進設定抽屜）

機電上限 15%

梯廳上限 5%

陽台上限 10%

單層屋突上限 12.5%

坪數換算 1 m² = 0.3025 坪

變動參數（可調）

預估公設比（預設 35%，合理區間提示 30%–40%）

車位單位面積（預設 30 m²/位）

折減係數（預設 0.85）

自動推估（顯示於本頁下方）

預估興建總樓高 = (最寬道路×1.5)+6

預估地上層數 = 預估興建總樓高 ÷ 預估地上單層樓高（預設採 ceil）

7.2 Step2：建築面積試算（Auto, Read-only）

輸入：

獎勵容積率（%）（預設 0，放在頁面上方可調）

自動計算（輸出 m² + 坪）：

最大建築面積 = 基地面積×建蔽率

法定容積面積 = 基地面積×容積率

獎勵容積面積 = 基地面積×容積率×獎勵容積率

機電空間面積 = (法定+獎勵)×15%

梯廳空間面積 = (法定+獎勵+機電)×5%

陽台空間面積 = (法定+獎勵+機電+梯廳)×10%

屋突面積 = 最大建築面積×12.5%×屋突層數

開挖面積 = 基地面積×開挖率

地下層面積 = 開挖面積×地下層數

總樓地板面積 = 法定+獎勵+機電+梯廳+陽台+屋突+地下層面積

7.3 Step3：銷售面積試算（Minimal input）

輸入（可調）：

車位單位面積（沿用變動參數）

折減係數（沿用變動參數）

一樓銷售面積覆寫（選填；若填則優先）

自動計算：

預估車位總面積 = 地下層面積×折減係數

預估車位總數量 = 車位總面積÷車位單位面積

一樓銷售面積 = [(總樓地板面積−車位總面積)×折減係數÷地上層數]（若覆寫則採覆寫）

二樓以上銷售面積 = 總樓地板面積−車位總面積−一樓銷售面積

總銷售面積 = 總樓地板面積−車位總面積

7.4 Step4：都市更新成本費用（Simplified）

設計原則：「預設全部有值」，避免地主卡在成本欄位。

必填最少

更新後預估總戶數（戶）

貸款年利率（%）（預設 2%~3% 提示）

權利變換費用（元，若未知可填 0）

鄰房鑑定戶數（若未知可填 0）

其餘成本費率採固定參數（可於進階抽屜調整，預設不讓使用者碰）：

設計費 9%

公共基金 0.4%

規費 0.1%

外審 0.01%

獎勵申請 0.2%

外接管線 97,500/戶

地籍整理 20,000/戶

印花稅 0.1% 等

輸出：

成本逐項表＋小計＋總計（預估總成本支出）

7.5 Step5：評估報告（核心輸出）

輸入：

銷售面積百分比（%）（用於變現，預設 20%）

預估車位單價（元/位）

預估一樓單價（元/坪）

預估二樓以上單價（元/坪）

輸出核心指標：

土地坪效

預估總銷售收入

共同負擔比

預估變現車位數量

預估變現銷售面積

預計變現總金額

分回比例

預估分回室內坪數

一坪換幾坪

並提供兩種模式切換（無 LLM）：

解釋模式：固定模板白話說明 + 本案數值

決策模式：規則門檻輸出等級（如 綠/黃/紅）+ 3 條原因 + 建議清單

10. 驗證規則（避免使用者卡住）

必填缺漏：直接指出欄位並跳轉定位

百分比：0–100

折減係數：0–1

面積/金額：不得為負

分母不得為 0（例如總銷售收入）

自動顯示「目前無法計算」原因，不得顯示錯誤堆疊

11. 技術限制（簡單環境）

不使用任何資料庫

不使用登入/帳號

所有計算以獨立模組 pure functions 實作

引用庫為靜態檔案

至少 5 組單元測試覆蓋正常與邊界情境
